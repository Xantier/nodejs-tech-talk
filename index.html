Node.js

It is not:

A web framework

A programming language

It wants to be:

Simple and relatively easy to understand and use

Useful for I/O based programs that need to be fast and/or handle lots of connections

It is designed to be good at:

  • Building network programs using the protocols of the web: HTTP, TCP, UDP, DNS and SSL
  • Building programs that read and write data to the filesystem or local processes/memory
    • Databases (e.g. MySQL, PostgreSQL, MongoDB, Redis, CouchDB)
    • APIs (e.g. Twitter, Facebook, Apple Push Notifications)
    • HTTP/WebSocket connections (from users of a web app)
    • Files (image resizer, video editor, internet radio)

    • Event-driven, Non-blocking I/O (web scale) model

  • Node.js core APIs

    • REPL
